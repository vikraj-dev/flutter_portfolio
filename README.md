# Flutter 3D Developer Portfolio

A modern, animated, dark-themed Flutter Web portfolio with pseudo-3D effects
— a mouse-tracking 3D tilt on project/skill cards, a gently floating 3D
avatar, and tap-to-flip 3D highlight cards. Built with pure Flutter (no
external 3D packages or model files), so it's lightweight and easy to host
on GitHub Pages.

## ✨ Features

- 6 "screens" (Home, About, Skills, Projects, Experience, Contact) inside a
  single responsive shell — works great on mobile, tablet and desktop.
- Animated splash screen, page transitions, typewriter role text, animated
  stat counters and staggered scroll-in animations.
- Pseudo-3D effects built with Flutter's `Matrix4`/`Transform` — no extra
  dependencies:
  - **Tilt3DCard** — real 3D tilt that follows your mouse on Project &
    Skill cards.
  - **Floating3D** — a gently rotating/floating 3D avatar.
  - **Flip3DCard** — tap or hover the "About" highlight cards to flip them
    in 3D.
- A working contact form that opens the visitor's email client (mailto)
  with the message pre-filled.
- One file to edit for all of your content: `lib/data/portfolio_data.dart`.

## 🚀 Getting started

This zip contains only the `lib/` folder and supporting config — not the
platform folders (`web/`, `android/`, `ios/`), since those are generated by
your local Flutter SDK and can vary by version. Follow these steps:

1. **Install Flutter** if you haven't already: https://docs.flutter.dev/get-started/install
2. **Create a fresh Flutter project** (this generates the correct platform
   folders for your installed SDK version):
   ```bash
   flutter create flutter_portfolio
   cd flutter_portfolio
   ```
3. **Copy the contents of this zip into that folder**, overwriting the
   generated `lib/` folder and `pubspec.yaml`. Also copy over
   `analysis_options.yaml`, `.gitignore` and the `.github/` folder if you
   want the auto-deploy workflow.
4. **Install dependencies**:
   ```bash
   flutter pub get
   ```
5. **Run it locally** in Chrome:
   ```bash
   flutter run -d chrome
   ```
6. **Personalize it** — open `lib/data/portfolio_data.dart` and replace the
   placeholder name, bio, email, social links, skills, projects and
   experience with your own.

## 🌐 Deploying to GitHub Pages

### Option A — GitHub Actions (recommended, fully automatic)

1. Push this project to a new GitHub repository.
2. Open `.github/workflows/deploy.yml` and replace `REPO_NAME` in the
   `--base-href "/REPO_NAME/"` line with your actual repository name.
3. In your repo, go to **Settings → Pages** and set **Source** to
   **GitHub Actions**.
4. Push to the `main` branch — the workflow will build and deploy
   automatically. Your site will appear at
   `https://<your-username>.github.io/<REPO_NAME>/`.

> Note: GitHub's official Action versions occasionally get superseded —
> if a step fails saying an action version is deprecated, just bump the
> version number in `deploy.yml` (e.g. `actions/checkout@v4` →
> `actions/checkout@v5`).

### Option B — Manual build & deploy

```bash
flutter build web --release --base-href "/REPO_NAME/"
```

Then push the contents of `build/web` to a `gh-pages` branch (or use a tool
like the `peaceiris/actions-gh-pages` action), and enable Pages on that
branch in your repo settings.
